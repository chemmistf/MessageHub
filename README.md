# Онлайн Мессенджер (Backend)

## Описание проекта
Данный проект представляет собой backend-систему для онлайн-чата и мессенджера, реализованную на платформе .NET с использованием языка C#. Система предназначена для коммерческого использования и обеспечивает масштабируемое и безопасное взаимодействие пользователей в режиме реального времени.

## Цели проекта
- Обеспечение надежного и высокопроизводительного обмена сообщениями между пользователями.
- Поддержка масштабирования для большого числа одновременно активных пользователей.
- Интеграция с системами управления контентом и внешними сервисами.
- Обеспечение безопасности данных и соблюдение стандартов конфиденциальности.
- Возможность расширения функционала для бизнес-процессов (платежи, уведомления, администрирование).

## Технологический стек
- Язык программирования: C#
- Платформа: .NET 8
- Фреймворк: ASP.NET Core
- База данных: PostgreSQL 
- ORM: Entity Framework Core
- Системы кэширования: Redis
- Сообщения в реальном времени: SignalR
- Логирование: Serilog 
- Документация API: Swagger 
- Контейнеризация: Docker
- CI/CD: GitHub 

## Архитектура
Проект построен на основе **Clean Architecture**, что обеспечивает:
- Разделение на слои: Domain, Application, Infrastructure, API.
- Независимость бизнес-логики от инфраструктуры и внешних зависимостей.
- Простоту тестирования и поддержки кода.
- Масштабируемость и возможность интеграции с другими сервисами.

## Документация API
- Все REST API и WebSocket-эндпоинты документированы с использованием **Swagger/OpenAPI**.
- Поддерживается автогенерация документации на основе аннотаций контроллеров и моделей.
- Примеры запросов и ответов доступны через Swagger UI.

## Обязательный функционал (Required Features)
- **Аутентификация**: регистрация, логин, JWT-токены, восстановление пароля.
- **Управление отелями**: создание, редактирование, удаление и просмотр информации об отелях.
- **Управление номерами**: CRUD операции для номеров с учетом типа, стоимости и доступности.
- **Поиск**: поиск отелей и номеров по различным фильтрам (местоположение, дата, цена).
- **Бронирование**: создание, изменение и отмена бронирований.
- **Платежи**: интеграция с платежными шлюзами, обработка транзакций.
- **Отзывы**: возможность оставлять и модераторить отзывы пользователей.
- **Уведомления**: push-уведомления и email для событий (подтверждение бронирования, новые сообщения).
- **Администрирование**: управление пользователями, ролями и правами доступа.
- **Логирование**: централизованное логирование действий пользователей и системных событий.
- **Безопасность**: защита от атак, шифрование данных, контроль доступа.

## Безопасность
- Использование JWT и OAuth2 для аутентификации.
- Шифрование конфиденциальных данных (AES, TLS).
- Защита от CSRF, XSS, SQL-инъекций и других веб-угроз.
- Регулярные аудиты логов и мониторинг активности.

## Тестирование
- Модульные тесты: xUnit / NUnit
- Интеграционные тесты: тестирование взаимодействия между слоями и внешними сервисами.
- Покрытие тестами основных бизнес-сценариев.
- CI/CD интеграция для автоматического запуска тестов при каждом коммите.

## Roadmap
- Поддержка мультиязычности интерфейса.
- Внедрение аналитики использования и метрик производительности.
- Расширение возможностей чата: групповые чаты, медиа-сообщения, стикеры.
- Интеграция с внешними мессенджерами и социальными сетями.
- Внедрение микросервисной архитектуры для отдельных компонентов (чат, платежи, уведомления).
